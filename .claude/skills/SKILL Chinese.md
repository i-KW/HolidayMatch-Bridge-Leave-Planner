# HolidayMatch · 假期对照 & 拼假规划

## 触发条件

用户要求制作两个国家的假期对照日历，或为 HolidayMatch 项目新增国家组合时触发。

---

## 工作流程

### 0. 时间判断 — 动态年份搜索与判断

识别用户表述中的时间线索，避免写死固定年份：

- **模糊表述识别**：识别"下周出行"、"下个月"、"最近"等模糊时间表述，通过 WebSearch 确认当前日期后推算目标月份
- **距离判断**：
  - 目标时间距今 **≤1 个月** → 默认今年，直接执行
  - 目标时间距今 **≥2 个月** → 主动询问"做今年的还是明年的日历？"
- **12 月跨年提示**：当前为 12 月时，主动提示"要不要连明年初的一起做？"
- 如果收到"明年/next year"的指令，年份自动 +1

> **目标**：对齐真实规划场景（临时出行 vs 提前计划），减少无效输出。

### 1. 并行调研假期数据 — 多源并发 + 交叉验证

> 各国独立并行检索 + 分源独立检索，不串行等待。

**并行维度**（同时进行）：
- **按国家并行**：Agent A → 国家 1 假期 / Agent B → 国家 2 假期
- **按来源并行**：对同一国家，同时搜索政府官网 + 第三方日历站 + timeanddate.com 等多源
- **按时间并行**：如需搜索多年（如今年+明年），年份之间也并行

**交叉验证机制**：
- 同一日期出现在多个来源 → ✅ 高置信度
- 仅单一来源 → ⚠️ 中置信度，标注来源
- 来源间冲突 → ❓ 标记为待确认，追加搜索

> **注意**：伊斯兰/农历节日（如开斋节、春节）每年公历日期不同，WebSearch 时必须指定年份，不能靠模型记忆推算。

> **优势**：速度 ×2，上下文干扰 ↓，准确性 ↑

### 2. 确定年份的标准日历

按 **公历（Gregorian Calendar）+ ISO 8601 周标准**生成该年 12 个月的月历底板，确保日期与星期正确对齐后，再填入节日数据。

| 项目 | 标准 |
|------|------|
| 历法 | 公历（Gregorian Calendar） |
| 每周起始 | **周一 → 周日**（与现有模板一致） |
| 月天数 | 1月31 / 2月28(闰29) / 3月31 / 4月30 / 5月31 / 6月30 / 7月31 / 8月31 / 9月30 / 10月31 / 11月30 / 12月31 |
| 闰年规则 | 公元年份能被 4 整除但不能被 100 整除，或能被 400 整除 |
| 周几计算 | 可用 WebSearch 查"{年份}年日历"或 {年份} calendar"获取全年的日-周对照，也可用已知算法（Zeller / Tomohiko Sakamoto）推算 |
| 验证手段 | WebSearch 搜索「{年份} calendar printable」对比验证 1 月 1 日和几个关键日期的星期是否正确 |

**重要**：先用月份天数和星期正确生成空月历表格，再往上填节日和标注。

### 3. 生成日历 HTML

严格按照现有模板的结构：

- **单文件 HTML** — 零依赖
- **颜色标记**：
  - A 国 = 绿色（`class="uk"` 或 `class="cn"`）
  - B 国 = 橙色/蓝色（`class="es"` 或 `class="ie"`）
  - 两国共同 = 紫色（`class="both"`）
  - 地区性 = 红色（`class="regional"`）
  - 调休上班/补班 = 红色底（`class="work"`）
  - 周末 = 浅蓝底（`class="weekend"`）
- **逐月表格**，对齐星期
- **单元格内标签**：emoji + 节日简称
- **月份标题**：汇总当月节日 + 两国共同标记（➡ X/X 🟣 两国同休）
- **英文为主**，适当加本地语言对照

### 4. 典型特殊规则与地区差异

生成日历和拼假攻略时必须考虑以下规则：

| 规则 | 适用国家 | 说明 |
|------|----------|------|
| **调休补班** | 🇨🇳 中国 | 节假日前后周末上班来凑长假，需标注"补班/调班" |
| **Substitute Bank Holiday** | 🇬🇧 英国 | 假日逢周末则顺延至下周一 |
| **Regional Holidays** | 🇪🇸 西班牙 | Jueves Santo / Lunes de Pascua 等各自治区不同 |
| **Puente（桥假）** | 🇪🇸 西班牙 | 假日落在周二/周四时，中间周一/周五常被用作"puente"请假连休 |
| **振替休日** | 🇯🇵 日本 | 假日逢周日则顺延至周一 |
| **国民の休日** | 🇯🇵 日本 | 前后两天都是假日时，中间工作日变为假日 |
| **대체공휴일** | 🇰🇷 韩国 | 假日逢周末则顺延至下一工作日 |
| **Pont（桥假）** | 🇫🇷 法国 | 假日逢周四可请周五连休，类似西班牙 puente |
| **Feiertage 联邦制** | 🇩🇪 德国 | 各州假日不同，无全国统一的"州假"列表 |
| **浮动节日** | 多国 | 伊斯兰节日（开斋节/宰牲节）、农历节日（春节/中秋）每年的公历日期不同，需 WebSearch 指定年份查询，不可记忆推算 |
| **联邦 vs 州假** | 🇺🇸 美国 | Federal holidays vs state observed holidays 不同 |

### 5. 验证兜底 — 置信度标记 + 来源透明

> 不伪装确定性，诚实暴露边界。

- **置信度标记**：
  - ✅ 高置信度：多源一致 + 政府官网确认
  - ⚠️ 中置信度：第三方来源一致但未查到官方文件
  - ❓ 待确认：来源冲突或仅有单一来源
- **来源透明**：每个日期旁标注来源链接或来源类型（gov / media / calendar site）
- **AI 生成标识**：拼假攻略部分明确标注"AI 生成建议，仅供参考"
- **免责声明**：浮动节日（伊斯兰/农历）的实际日期可能因月相观测或官方宣布而有±1天差异

> **目标**：让用户知道信息的可靠程度，而不是把所有信息伪装成同等确定的。一个诚实的"不确定"比一个虚假的"确定"更有价值。

### 6. 汇总部分
- 每国年度假日汇总表（含调休/补班天数）
- **共同放假日对照表**（标 🟣）
- 地区性差异说明

### 7. 拼假指南
计算最优年假组合方案：
- 每个节日集群：请几天假 → 连休几天
- **最高亮**：`style="background:#fff8e1;"` 标记连休最长的那行
- 格式："请 X 天假 → Y 天连休"
- 酌情纳入两国的假期交叉建议（如 A 国放假 B 国不放假时，A 国人可以飞去找 B 国人玩）

### 8. 数据来源 & 页脚
- 链接官方来源
- 最后更新日期（用 WebSearch 获取当前日期）
- 地区性差异 / 浮动节日免责声明

## 产品理念
- **并排对照** — 一眼扫过就知道两国假期
- **实用至上** — 拼假攻略才是核心价值，不只是日历
- **零摩擦** — 单 HTML，不用安装，打印友好
- **易分享** — 直接丢群聊或家庭共享文档
